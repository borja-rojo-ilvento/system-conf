# `lib/hardware/`

Per-machine hardware specifications.

## Layout

- `machines/<machine-name>/`
  - `hardware-configuration.nix` — generated by `nixos-generate-config`. Don't
    hand-edit; rerun the generator if hardware changes.
  - `default.nix` — machine-specific knobs that don't belong in the
    auto-generated file: GPU drivers, hybrid graphics setup, Bluetooth radio,
    lid/power button policy, firmware blobs, vendor-specific quirks.

A "machine" here is a physical box. The same machine directory can be reused
by different `nixosConfigurations` if its system shape changes — e.g. the
desktop hardware (`bung-box`) might host a `everest` workstation today and a
`bung-box-server` host later.

## What goes here vs `lib/system/`

| Belongs here                                         | Belongs in `lib/system/`                          |
|------------------------------------------------------|---------------------------------------------------|
| `hardware.nvidia.*`, `hardware.bluetooth.*`          | `services.pipewire`, `services.printing`          |
| `boot.kernelModules`, `boot.initrd.*`                | `xdg.portal`, `services.dbus`                     |
| Lid/power button policy that varies per chassis      | Default lid/power policy (overridable per host)   |
| `services.xserver.videoDrivers` overrides            | Display manager + desktop environment selection   |

Rule of thumb: if swapping the motherboard would invalidate the line, it
belongs here.

## Adding a machine

1. Boot into the new hardware with a NixOS installer.
2. `nixos-generate-config --root /mnt --dir <repo>/lib/hardware/machines/<name>`.
3. Hand-write `default.nix` next to it for the items above.
4. If `nixos-hardware` has a profile for the machine, import it from
   `default.nix` and treat its defaults as the baseline — don't blindly
   override `videoDrivers` etc.; check what the upstream profile already sets.
